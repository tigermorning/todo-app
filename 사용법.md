# 사용법

이 문서는 실제로 이 Todo 앱을 쓰는 방법을 한국어로 정리한 안내서입니다. 설치·개발 관련 내용(파이썬 설치, 의존성 설치 등)은 [README.md](README.md)를 참고하세요. 여기서는 "이미 서버가 떠 있다"는 가정 하에 각 기능을 어떻게 쓰는지만 다룹니다.

## 1. 실행하기

Windows에서는 `todo-app` 폴더의 **`start.bat`**을 더블클릭하면 됩니다. 검은 창이 뜨고 `Application startup complete.`가 보이면 준비 완료입니다. 이 창을 닫으면 서버도 같이 꺼지니, 쓰는 동안은 창을 열어두세요.

브라우저에서 **http://localhost:8000** 을 열면 앱이 보입니다.

> 서버가 갑자기 안 뜨거나 "사이트에 연결할 수 없음"이 자꾸 뜬다면, 이미 다른 창에서 서버가 떠 있어서 포트(8000번)가 겹치는 경우가 대부분입니다. 열려 있는 다른 `start.bat` 창을 전부 닫고 하나만 새로 실행하세요.

## 2. 빠른 입력 — 자연어로 바로 추가하기

맨 위 입력창에 한글 문장을 그대로 치고 Enter를 누르면, 제목과 날짜/시간을 자동으로 뽑아냅니다.

| 입력 | 결과 |
|---|---|
| `내일 오후 1시에 네일숍` | 제목 "네일숍", 마감 내일 13:00 |
| `20:30 회의` | 제목 "회의", 마감 오늘 20:30 |
| `다음주 화요일에 병원` | 제목 "병원", 마감 다음주 화요일 |
| `7월 25일에 생일파티` | 제목 "생일파티", 마감 7월 25일 |
| `매일 9시 약 복용` | 매일 반복(09:00) — 언제까지 반복할지 추가로 물어봄 |
| `8/3-11/5 격주 공원 산책` | 8/3~11/5까지 2주마다 반복 |
| `5/1-9/30 격주로 산책, 9월 첫째주 제외` | 위와 같되 **9월 첫째주는 건너뜀** |
| `수요일마다 산책` | 매주 수요일 반복 — 언제까지 반복할지 추가로 물어봄 |

### 반복이 되려면 반드시 "트리거 단어"가 있어야 함

`화요일에 병원`처럼 요일만 딱 언급하면 **이번 한 번**의 약속으로 처리됩니다(매주 반복 아님). 실제로 대부분의 요일 언급은 "이번 주만"이라는 뜻이기 때문에 일부러 이렇게 설계했습니다. 반복시키려면 다음 중 하나를 꼭 포함하세요: `매일`, `매주`, `격주`, 날짜 범위(`M/D-M/D`), 또는 `{요일}요일마다`(예: `수요일마다`).

### 반복 일정에서 특정 기간 제외하기

빠른 입력 문장 끝에 `, <기간> 제외`를 붙이면 별도 화면 없이 바로 제외 기간을 지정할 수 있습니다. 두 가지 표현을 인식합니다.

- **날짜 범위 직접 지정**: `M/D-M/D 제외` 또는 `M월 D일부터 M월 D일까지 제외`
- **몇째 주로 지정**: `M월 첫째주 제외` / `둘째주` / `셋째주` / `넷째주` / `다섯째주` 제외 (첫째주 = 그 달 1~7일, 둘째주 = 8~14일, ...)

한 문장에는 제외 구간을 하나만 넣을 수 있습니다. 여러 구간을 빼려면 아래 "상세 입력" 폼의 "제외 기간 추가" 버튼을 여러 번 눌러 추가하세요.

문장 안에 이미 등록된 카테고리 이름(예: "필라테스")이 그대로 들어있으면 자동으로 그 카테고리가 붙고, 없으면 "기타"로 들어갑니다 — 나중에 목록에서 배지를 클릭해 바로 바꿀 수 있습니다.

## 3. 상세 입력 — 폼으로 직접 입력하기

"상세 입력"을 누르면 제목/카테고리/날짜·시간/반복 설정을 각각 따로 채우는 폼이 펼쳐집니다. 자연어를 안 쓰고 정확하게 지정하고 싶을 때 이쪽을 쓰세요.

> **주의**: 여기 제목 칸은 입력한 그대로 저장됩니다(자연어 해석 안 함). "수요일 산책, 9월 첫째주 제외"라고 타이핑하면 그게 제목 그대로 저장되고, 실제 날짜/반복/제외는 아래 별도 필드로 지정해야 합니다.

## 4. 카테고리 관리

기본 카테고리 6개(🏃 운동, 💼 업무, 📚 공부, 🧹 집안일, 👥 약속, 📌 기타)가 있고, 각각 정해진 색 그룹에 속합니다. 같은 그룹에 속한 카테고리는 배지 색이 항상 같아서, 이름이 달라도 한눈에 "이건 운동 계열이구나"를 알 수 있습니다(예: "필라테스"를 운동 그룹에 넣으면 기존 운동 카테고리와 같은 파란색 배지가 됨).

- **새로 만들기**: 카테고리 드롭다운에서 "+ 새 카테고리" 선택 → 이름/아이콘(이모지, 선택) 입력 → 색 그룹 선택 → 저장
- **이름·아이콘·그룹 수정**: 드롭다운에서 "⚙ 카테고리 관리" 선택 → 목록에서 원하는 줄을 고쳐서 "저장" 클릭
- **삭제**: 같은 관리 화면에서 "삭제" 클릭. 그 카테고리를 쓰던 할 일은 삭제되지 않고 "카테고리 없음" 상태가 됩니다.
- **할 일 하나만 카테고리 바꾸기**: 목록에서 그 할 일의 카테고리 배지를 클릭하면 바로 그 자리에서 드롭다운이 뜹니다.

## 5. 검색과 필터

목록 위 검색창은 제목을 실시간으로 필터링합니다. 옆의 카테고리 드롭다운은 하나의 카테고리만 보거나("전체 카테고리"로 전부 보기) 고를 수 있습니다. 사이드바 달력에서 날짜를 클릭해 그 날짜로 좁힌 상태에서도 검색/카테고리 필터가 같이 적용됩니다 — 다시 전체 날짜를 보려면 "전체 보기"를 먼저 누르세요.

## 6. 다중 선택 — 여러 개를 한꺼번에

**제목이나 시간 텍스트를 클릭해야** 선택이 시작됩니다(체크박스를 그냥 클릭하면 평소처럼 완료 체크만 됩니다).

- **Shift+클릭**: 다른 항목의 제목/시간/체크박스를 Shift 누르고 클릭하면, 마지막 선택 지점부터 그 항목까지 연속으로 다 선택됩니다(탐색기/파인더처럼).
- **전체 선택**: 목록 위 버튼 — 지금 화면에 보이는 항목(검색/필터/날짜 적용된 상태 그대로) 전부를 한 번에 선택.
- 하나라도 선택되면 아래에 일괄 작업 바가 나타나서 **선택 삭제** 또는 **선택 해제**(삭제 없이 선택만 풀기)를 할 수 있습니다.
- **선택된 상태에서 체크박스를 누르면 선택된 항목 전체**의 완료 상태가 한꺼번에 바뀝니다. 아무것도 선택 안 된 상태에서 체크박스만 누르면 그 항목 하나만 바뀝니다(평소 동작).

## 7. 사이드바

### 달력

마감이 있는 날에 점이 표시됩니다. 날짜를 클릭하면 그 날짜로 목록이 좁혀지고, "전체 보기"를 누르면 다시 전체가 보입니다.

### Tracker (습관 트래커)

"＋" 버튼으로 원하는 이름의 트래커를 몇 개든 만들 수 있고, 드롭다운으로 전환합니다. 세 가지 방식이 있습니다.

1. **수동 (기본값)**: 날짜 칸을 클릭할 때마다 빈칸 → ○(완료, 초록) → ✕(실패, 빨강) → 빈칸 순으로 바뀝니다. "⧉" 버튼으로 복제할 수 있고(기존 기록을 복사할지 선택 가능), 이름은 자유롭게(운동, 독서, 금주 등).
2. **반복 일정과 연동**: 만들 때 드롭다운에서 이미 등록된 반복 일정(예: "매일 운동")을 고르면, 그 반복 일정의 실제 완료 여부로 날짜 칸이 **자동으로** 채워집니다(체크하면 ○, 그 날이 지났는데 안 했으면 자동 ✕). 직접 클릭해서 수정하거나 복제할 수 없습니다 — 데이터가 전부 연결된 할 일에서 나오기 때문입니다.
3. **주간 목표**: "매일"이 아니라 "매주 두 번"처럼 주 단위 목표를 추적할 때 씁니다. 만들 때 "주간 목표로 추적"을 체크하고 목표 횟수(1~7)를 정하면 됩니다. 날짜 칸을 클릭하면 그날 했다는 표시로 **동그라미 테두리**가 켜지고(완료/실패 구분 없이 했는지 안 했는지만), **한 주(일~토) 동안 목표 횟수를 채우면 그 주 전체 칸의 배경색이 강조**됩니다. 반복 일정 연동과는 동시에 쓸 수 없습니다(둘 중 하나만).

> 월 경계(예: 7월 마지막 주가 8월로 이어지는 경우)는 지금 보고 있는 달의 날짜만으로 목표 달성 여부를 계산합니다 — 화면에 보이는 만큼만 정확하다고 생각하면 됩니다.

## 8. 지난 할 일 확인

탭을 열어둔 동안 60초마다 마감이 지났는데 아직 체크 안 된 할 일이 있는지 확인해서 배너로 띄웁니다. 완료 처리, 다시 일정 잡기, 몇 시간 뒤로 미루기 중 하나를 고를 수 있습니다. "🔔 알림 켜기"를 한 번 누르면 이 상황에서 브라우저 알림도 같이 뜹니다(탭이 열려 있어야만 동작 — 꺼두면 알림도 안 옵니다).

## 9. 구글 캘린더 연동 (선택)

본인 구글 계정으로 직접 로그인/동의해야 하는 단계라 대신 해줄 수 없는 부분입니다.

1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트를 만들고 **Google Calendar API**를 사용 설정
2. OAuth 동의 화면 설정(개인용은 "외부" 유형으로 충분, 테스트 사용자로 본인 이메일 추가)
3. **OAuth 클라이언트 ID**(데스크톱 앱 유형)를 만들어 JSON을 `credentials.json`으로 이 폴더에 저장
4. `.env.example`을 `.env`로 복사
5. 본인 터미널에서 한 번만 실행 (브라우저가 열리며 로그인/동의 창이 뜸):
   ```bash
   python setup_google_calendar.py
   ```
6. 서버 재시작 → 다가오는 일정이 "제안된 할 일"로 나타남. "추가"를 누르면 그 하나만 할 일로 저장되고, "무시"를 누르면 이후로 다시 안 뜹니다.

### 여러 개를 한꺼번에 수락하기

각 제안 항목에 체크박스가 있습니다. 여러 개 체크하거나 "전체 선택"을 누른 뒤, 아래 나타나는 "선택 추가" 버튼을 누르면 체크한 것들이 한꺼번에 할 일로 저장됩니다. "선택 해제"는 체크만 풀어줍니다(수락 안 함).

### 다른 계정의 캘린더도 같이 보기

지금 로그인된 구글 계정에서, [calendar.google.com](https://calendar.google.com) 좌측의 "다른 캘린더" 목록에 다른 계정 캘린더를 공유받거나 구독해서 추가하고 체크박스를 켜두면, 그 캘린더의 일정도 여기 제안 목록에 같이 뜹니다(어느 캘린더에서 온 건지 이름 배지로 표시됨). 구글 캘린더 쪽에서 체크박스를 끄면 여기서도 안 보이게 됩니다.

## 10. 알려진 한계

- 자연어 빠른 입력은 한국어만 이해합니다. 상세 입력 폼의 날짜/시간 선택기는 언어와 무관하게 동작합니다.
- 지난 할 일 알림은 브라우저 탭이 열려 있을 때만 옵니다 — 항상 켜져 있는 별도 백그라운드 서비스는 없습니다.
- 종료일 없는 반복 일정은 60일치씩만 미리 만들어두고 서버가 켜질 때마다 자동으로 더 채웁니다(종료일이 있는 반복은 그 즉시 전부 만들어짐).
- 빠른 입력의 반복 제외 구문은 문장당 한 구간만 인식합니다. 여러 구간을 빼려면 상세 입력 폼을 쓰세요.
- 주간 목표 Tracker는 달 경계에 걸친 주를 그 달 안의 날짜만으로 판정합니다(§7 참고).
- 로그인·비밀번호가 없습니다 — 지금은 이 컴퓨터(또는 같은 와이파이) 안에서만 쓰는 걸 전제로 만들어졌습니다.

더 자세한 설계 배경과 결정 이유가 궁금하면 [PRD.md](PRD.md)를 참고하세요.
